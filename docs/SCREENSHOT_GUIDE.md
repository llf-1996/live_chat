## 🛠️ 截图工具推荐

### Windows
- **Snipping Tool** / **截图工具**（系统自带）
  - 快捷键：`Win + Shift + S`
- **ShareX**（免费开源）
  - 支持截图、标注、上传
  - 下载：https://getsharex.com/

### macOS
- **截图工具**（系统自带）
  - 全屏：`Cmd + Shift + 3`
  - 选区：`Cmd + Shift + 4`
  - 窗口：`Cmd + Shift + 4 + 空格`
- **CleanShot X**（付费，功能强大）

### 跨平台
- **Lightshot**
  - 下载：https://app.prntscr.com/
- **Flameshot**（开源）
  - 下载：https://flameshot.org/

## ✂️ 截图后处理

### 1. 裁剪和缩放
- 保持宽高比
- 建议宽度：800-1200px
- 去除不必要的空白区域

### 2. 添加标注（可选）
- 使用箭头、方框突出重要功能
- 添加文字说明
- 推荐工具：ShareX、Snagit、Photoshop

### 3. 压缩
- 使用 [TinyPNG](https://tinypng.com/) 压缩
- 目标大小：200-500KB/张
- 保持清晰度

### 4. 格式
- **推荐格式**：PNG（清晰度高）
- **备选格式**：JPG（文件更小）
- **避免格式**：BMP、TIFF（文件过大）

## 📤 保存和提交

### 1. 保存位置
将截图保存到项目的 `docs/screenshots/` 目录：

```
docs/
└── screenshots/
    ├── login.png
    ├── buyer-chat-list.png
    ├── buyer-chat-window.png
    └── ...
```

### 2. 文件命名
严格按照 `docs/screenshots/README.md` 中的命名规范。

### 3. Git 提交
```bash
cd D:\workspace\live_chat
git add docs/screenshots/*.png
git commit -m "docs: 添加界面截图"
```

## 💡 提示

1. **测试数据准备**：
   - 在截图前，准备好充足的测试数据
   - 创建多个用户、会话、消息
   - 确保数据真实且有代表性

2. **浏览器窗口**：
   - 截图前关闭浏览器开发者工具
   - 隐藏书签栏（Ctrl+Shift+B）
   - 使用全屏模式或固定窗口大小

3. **隐私保护**：
   - 不要包含真实的敏感信息
   - 使用测试账号和模拟数据

4. **一致性**：
   - 使用相同的浏览器和分辨率
   - 保持相同的 UI 主题
   - 统一的截图风格
